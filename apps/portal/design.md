# RNFleetManager Portal — Design System

This document defines the visual language for the **RNFleetManager Network Operations Console**
(`apps/portal/src/web/index.html`). It exists so future UI work stays consistent with the
"modern network device management tool" look and feel.

> Scope: portal UI only. Core product logic (`apps/*`, `packages/*`) is unaffected by these
> design rules.

---

## 1) Design Principles

1. **Operations-grade dark UI** — a NOC/console aesthetic optimized for at-a-glance fleet health.
2. **Status is visual first** — every device/tunnel/apply state is encoded with color + motion,
   never plain text alone.
3. **Calm motion** — animation communicates state (in-progress, live, applied), never decoration
   for its own sake. Honor `prefers-reduced-motion`.
4. **Monospace for machine data** — IPs, MACs, timestamps, versions use a mono font so columns
   align and values are scannable.
5. **Stable layout** — rows are sorted by a stable key (`deviceId`, numeric-aware) so live
   auto-refresh never reorders/jerks the table.

---

## 2) Color Tokens

All colors are CSS variables on `:root`. Use tokens, never hard-coded hex, in new UI.

### Surfaces
| Token | Value | Use |
|---|---|---|
| `--bg` | `#0b1220` | App background base (with layered radial glows) |
| `--surface` / `--surface-2` | `#111a2e` / `#16213a` | Card gradients |
| `--surface-3` | `#1c2945` | Inputs, table header, progress track |
| `--line` / `--line-strong` | `rgba(148,163,184,.14/.28)` | Borders |

### Text
| Token | Value | Use |
|---|---|---|
| `--text` | `#e8eef9` | Primary text |
| `--text-dim` | `#9fb0cc` | Secondary/labels |
| `--muted` | `#6b7a96` | Placeholders, timestamps |

### Brand / Accent
| Token | Value | Use |
|---|---|---|
| `--accent` | `#5b8cff` | Primary brand (indigo) |
| `--accent-2` | `#22d3ee` | Secondary (cyan), code, highlights |
| `--accent-grad` | indigo→cyan 135° | Brand mark, primary buttons, card top-rule |

### Semantic status
| State | Token | Color |
|---|---|---|
| OK / online / up / applied | `--ok` | `#34d399` (green) |
| Warn / degraded / pending / restarting | `--warn` | `#fbbf24` (amber) |
| Bad / offline / down / failed | `--bad` | `#f87171` (red) |
| Neutral / unknown | `--neutral` | `#94a3b8` (slate) |

Each semantic color has a matching `*-soft` token for pill/badge backgrounds.

---

## 3) Typography

- **UI font:** `Inter` (Google Fonts), fallback `Segoe UI, Arial`.
- **Mono font:** `JetBrains Mono` (`--mono`) for IPs, MACs, versions, timestamps, audit JSON.
- Weights: 800 (brand/stat values), 700 (headings/buttons), 600 (labels/pills), 400–500 (body).
- App title 18px; section headings 17px; table body 13.5px; mono data 12–12.5px.

---

## 4) Components

### App bar (`.appbar`)
Sticky, blurred translucent slate bar with gradient **brand mark** (network-rack glyph) +
title/subtitle on the left, and `pill-meta` chips (API base, PSK header) on the right.

### Fleet stat cards (`.stats > .stat`)
Four KPI cards: **Total Devices, Online, LAN Applied, Errors**. Each has a tinted rounded icon
tile (`.s-ico.total|online|applied|error`) + large value + label. Updated by `updateFleetStats()`
on every device refresh.

### Cards (`.card`)
Gradient surface, 16px radius, soft shadow, 2px accent-gradient top rule, `rise` entrance
animation. Each card opens with a `.section-head` (icon tile + title + optional `.sub`).

### Status pills (`.pill`)
Generated by `statusPill(value)` in JS. Maps values to `pill-ok | pill-warn | pill-bad |
pill-neutral`. OK pills include a **pulsing dot** (`ping` animation) to signal "live/healthy".
Used for device **Status** and **Tunnel** columns.

### LAN Apply progress (`.lan-apply-*`)
Per-row animated bar driven by `normalizeLanApplyStatus(device)`:
- `applied` → solid green, 100%.
- `applying` / `pending` / `prepare` / `commit` → **animated indigo→cyan barber-pole stripes**.
- `failed` / `rolled_back` → red stripes with pulse.
- `unknown` → empty track.
Label shows a status dot + text. Status is inferred from `criticalChange.status`, an in-flight
push, or a `critical_lan_*` value in `lastError`.

### Push status ring (`.push-status`, `.ring-spinner`)
On **Push Device Config**, a **circular spinner** + live message box appears and the button is
disabled. Spinner turns solid green (`success`) on applied, red (`failed`) on error. While a push
is in flight the device table polls every **2s** (otherwise **5s**) via `setAutoRefresh()`.

### Buttons
- Primary: accent-gradient fill, dark text, lift on hover, leading SVG icon.
- `.ghost`: subtle surface + border, accent on hover (used for Refresh actions).
- Disabled: 55% opacity, no transform/shadow.

### Table
Dark, sticky uppercase header, hover row tint, fade-in rows, mono data cells, `count-badge` for
route count + config version, `err-text` (red mono) for errors, em-dash placeholder for empty.

### Endpoint cells (`.endpoint`, `.speed-chip`)
The **WAN (Interface / IP)** and **LAN (Interface / IP)** columns stack the interface name
(`.ep-if`, cyan mono) + a **link-speed chip** (`.speed-chip`) above the IP address. Speed is
formatted to real-world bandwidth by `formatSpeed(mbps)`: `100 Mbps`, `1 Gbps`, `10 Gbps`,
`100 Gbps`, `1 Tbps`. Per-interface speed is resolved via `interfaceSpeedMbps(d, name)`; speed is
shown only on these endpoint cells (removed from the Adapters detail column to avoid duplication).

### Icons
Inline **stroke SVGs** (`stroke-width="2"`, `currentColor`) — no icon font/CDN dependency.
Recurring glyphs: network rack (device/brand), gear (settings), tunnel, file-check (job),
audit/clock, signal (API), lock (PSK), refresh, up-arrow (push).

## 5) Motion

| Animation | Use | Notes |
|---|---|---|
| `rise` | card/stat entrance | 0.5s ease-out, translateY |
| `fadeIn` | table rows | 0.35s |
| `ping` | live/OK status dots | infinite pulse ring |
| `lanProgressMove` | LAN apply stripes | barber-pole motion |
| `lanProgressPulse` | failed apply | opacity pulse |
| `spinRing` | push spinner | rotation |
| `blink` | applying label dot | step blink |

All motion is disabled under `@media (prefers-reduced-motion: reduce)`.

---

## 6) Layout & Responsiveness

- Max content width `1680px`, generous padding.
- Stats grid: 4 cols → 2 cols (`≤1100px`) → 1 col (`≤620px`).
- Form fields flex and wrap; collapse to full width on narrow screens.
- Device table scrolls horizontally (`min-width: 1280px`) to preserve column integrity.

---

## 7) Rules for Future UI Work

- Reuse tokens and existing component classes; do not introduce new ad-hoc colors.
- Any new device/tunnel/job state must render through `statusPill()` or a semantic token.
- Keep the device table sort stable (`deviceId`, numeric-aware) to avoid refresh jerk.
- Prefer inline stroke SVGs over icon fonts/images (offline-friendly, themeable via `currentColor`).
- Encode async/long-running actions with a spinner + live status + auto-refresh, like the
  device config push flow.
